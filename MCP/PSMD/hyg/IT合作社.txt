密：公开
IT合作社章程
（临时）
	
总则
    一、 IT合作社是根据合作原则建立的，以满足IT相关专业人士在生活品质、个人发展的共同需求为宗旨的，自愿组成的联盟。
    二、 本章程按照下列原则制订：
        （一） 把管理问题转化为IT问题，便于社员参与管理和决策；
        （二） 依托宿主单位处理对外事务；
        （三） 在合作社层次借鉴其它行业合作社的经验，实行民主管理；
        （四） 在项目层次吸收IT行业成功经验，保证项目的进度与质量。
    三、 宿主单位代管对外事务，包括但不限于：税、发票、资产托管、知识产权托管、签订合同、材料采购、产品运营、客户服务、行政财务等。合作社注册独立企业时，托管于宿主单位的知识产权和资产转至新设立的企业。

合作社
    四、 加入本合作社须履行以下程序：
        （一） 只接受自然人的社员申请；
        （二） 提交书面申请，承诺遵守章程规定的各项义务，承诺按章程规定投入工时，承认合作社已签订的合同效力；
        （三） 经本合作社社员会议审核并讨论通过。
        （四） 以现金方式缴纳出资额三千元。
    五、 宿主单位以法人身份有权列席社员会议，但无表决权。
    六、 社员享有下列权利：
        （一） 参加社员会议，拥有选举权、被选举权，按照章程规定对本社实行民主管理；
        （二） 利用本合作社提供的各项公共服务和公用设备设施；
        （三） 按照章程规定或者社员会议决议分享盈余；
        （四） 查阅本社的章程、成员名册、社员会议记录、合作社收支记录、所有专户余额、专户间转账流水记录；
        （五） 自由提出退出申请，依照本章程规定退出本合作社。
    七、 本合作社成员须履行下列义务：
        （一） 遵守本合作社章程和各项规章制度，执行社员会议的决议；
        （二） 滚动公布六个月内的各周可投入工时（可以调整四个月以后的），按照约定参加工作；
        （三） 不从事损害本合作社成员共同利益的活动； 
    八、 社员有下列情形之一的，终止其社员资格：
        （一） 主动声明退出的；
        （二） 丧失民事行为能力的；
        （三） 被本合作社除名的。
    九、 本合作社社员的权利均不可转让、不可继承。如社员死亡，不再参与项目盈余分配，其社员专户资金转入合作社公益金。
    十、 社员有下列情形之一的，经管理员、监事共同查实予以除名：
        （一） 不遵守本合作社章程、不执行社员会议决议，不履行成员义务；
        （二） 给本合作社名誉或者利益带来严重损害的。	
    十一、 主动声明退出的社员须在季度终了的一个月前向管理员提出书面申请。成员资格自季度结束时终止，入社出资从合作社运营金专户退还到社员专户，如合作社运营金专户金额不足则在以后资金充裕时退还，具体时间有管理委员会决定。期间如合作社、项目出现亏损，仍从其社员专户中扣除其应分担的亏损金额。
被除名的社员，根据责任从其社员专户扣除其应承担的款项。被除名的社员，不退还入社出资。
主动声明退出的、被除名的社员，如果社员在项目期间声明退出，有约定工作没有完成的，不得参与该项目盈余分配，社员专户余额转入项目专户作为补偿。同时参与多个项目的由项目经理商定分配方式，分配前十天仍无法达成一致的，由管理员决定分配方案。
    十二、 本合作社的机构由社员会议、管理委员会、监事会构成。
        （一） 每累计八十张内部转账单、外部转账单，或者累计四十万元收入，或者累计登记八百个工时，或者上个管理周期结束一年，作为一个管理周期。每个管理周期结束，管理委员会提交一份书面工作报告、一份工作计划，并在此后最近一次社员会议上进行工作简报。
        （二） 每累计二十张内部转账单、外部转账单，或者累计十万元收入，或者累计登记二百个工时，或者上个监事周期结束一个季度，作为一个监事周期。每个监事周期结束，监事进行一次内部检查，对监事周期内合作社是否严格遵照章程运营，出具书面报告。对违规行为提出处理意见。
        （三） 管理员、监事的工作，可以任选项目登记工时。
    十三、 本合作社设有召集人、管理员、监事三个职务，均由社员会议选举，任期为三个管理周期，可以连选连任。
    十四、 社员会议是本合作社的最高权力机构，由全体社员组成。社员会议行使以下权利：
        （一） 选举和罢免召集人、管理员、监事；
        （二） 修改本章程及管理制度；
        （三） 决定临时的分配；
        （四） 批准所有对外合同；
        （五） 对本合作社的独立、解散、清算作出决议。
    十五、 管理委员会由管理员领导，执行社员会议的决议，管理未托管于宿主单位的事务。管理委员会其他成员由管理员任命，交社员会议备案。
        （一） 管理委员会每年安排四十次以上集合，应提前三天通知全体社员。参加的社员可以集中处理项目工作，或自由交流聚会。
    十六、 管理委员会有权征用一周之内的闲余工时（社员投入到合作社的工时减去为项目保留的）。
        （一） 工作：一般是合作社的公共服务，比如合作社网站的开发，对外宣传等工作。
        （二） 费用：参照召集人、管理员、监事三个职务的费用，要么都是义务工作，要都支付一定费用，根据工作内容由合作社运营金、公积金支付。
    十七、 监事会设监事两名，负责监督本章程以及依照本章场制定的规章执行。
    十八、 工时管理条款
每名社员对应三个工时数字：
        （一） 可投入工时：约定可以归合作社调用、还没有归属的工时，分到各周；由管理委员会负责收集和公布。
        （二） 项目保留工时：社员与项目经理约定后，保留给项目调用的工时，同时应在可投入工时减少同样数量的工时；可以同时在多个项目中保留工时；由管理委员会负责收集和公布。
        （三） 登记工时：由项目经理或合作社管理员审核工作结果符合要求后，根据合作社指定方式登记对应工时。
    十九、 责任承担：
        （一） 经社员会议表决通过后以某社员名义对外签订的合同，责任由合作社承担；
        （二） 因为某社员渎职、失误所造成的责任由该社员承担，所造成的损失偿付顺序是：责任人社员专户、合作社公益金、全体社员专户（按比例分担）、各项目专户（按比例分担）、合作社公积金、其它专户（按比例分担），如果仍然不足则合作社破产清算；
        （三） 因为某项目工作失误的责任由项目经理承担，所造成的损失偿付顺序是：项目经理社员专户、项目风险金、项目创始人社员专户、项目专户、项目成员社员专户、合作社公益金、全体社员专户（按比例分担）、其它项目专户（按比例分担）、合作社公积金、其它专户（按比例分担），如果仍然不足则合作社破产清算。
        （四） 没有特定社员、项目承担的责任由合作社集体承担，所造成的损失偿付顺序是：管理员社员专户、监事社员专户、召集人社员专户、合作社公益金、合作社公积金、全体社员专户（按比例分担）、全体项目专户（按比例分担）、其它专户（按比例分担），如果仍然不足则合作社破产清算。


议事规则
    二十、 社员会议
        （一） 社员表决权：
            1、 在总工时少于等于100时，所有社员拥有表决权。
            2、 在总工时大于100且小于500时，6个月内累计工时超过16工时的社员拥有表决权；
            3、 在总工时大于500时，6个月内累计工时超过40工时的社员拥有表决权。
            4、 所有拥有表决权的社员，表决权相等。
            5、 管理员不受工时限制，拥有表决权。
（工时是指已经使用、被登记的工时，包括被项目登记或者合作社管理员登记。总工时是指整个合作社已登记的工时。）
        （二） 社员会议由召集人发起，由管理员通知全体社员，会议通知应提前三天在主页公布，同时以email发给所有社员并且在QQ群中通知；
        （三） 出现以下情况之一，可以在三天内召开临时社员会议：
            1、 由三分之一社员提出；
            2、 监事会或管理委员会提出。
        （四） 社员会议应有三分之二以上有表决权的社员、至少一名监事会成员出席；
        （五） 社员会议由召集人指定主持人，由监事会指定记录员，没有指定时由出席的社员选出；临时社员会议由出席社员选举主持人和记录员；
        （六） 社员会议包括在线、当面两种召开模式，社员会议表决均为记名方式；
        （七） 社员会议议事过程按照《罗伯特议事规则第十版》进行；
        （八） 修改本章程、合作社的独立、解散、清算需要三分之二有表决权的社员同意，其他决议需要出席的、有表决权的社员二分之一以上同意；
        （九） 与宿主单位相关的条款，必须经过宿主单位同意方可修改；
        （十） 以任何社员名义对外签订合同的，必须经过该社员同意方才有效；
        （十一） 已经对外签订合同的，社员会议决议不得与之冲突；
        （十二） 会议记录电子版应公布在主页，表决票的实物应保留至少60天备查。
        （十三） 每次社员会议中管理员公布上次会议至今的所有合作社资金总表变化，由监事核实。
    二十一、 项目会议没有固定权利，项目创始人、项目经理可将自己的部分权利委托项目会议行使，委托后不得收回：
        （一） 项目成员是指该季度参与项目的社员以及项目创建人，项目会议由全体项目成员组成；
        （二） 临时项目成员是指该季度参与项目的非合作社成员，临时项目成员有参与项目会议的权利，但没有表决权。监时项目成员对项目有一定的知情权，具体知情权由项目经理决定。
        （三） 项目会议成员拥有相同的表决权；
        （四） 项目创始人、项目经理可以召集项目会议，并指定主持人，记录人由项目成员选举；
        （五） 项目会议应由应有三分之二以上项目成员出席；
        （六） 出席的项目成员二分之一以上同意，即可通过项目会议决议；
        （七） 项目会议包括在线、当面两种召开模式，项目会议表决均为记名方式；
        （八） 项目会议议事过程按照《罗伯特议事规则第十版》进行；
        （九） 会议记录电子版应当天报送管理委员会、全体项目成员，表决票实物交由管理委员会，应保留至少60天备查。

分配制度
    二十二、 本合作社采用按工时加权的分配制度。
        （一） 项目成员提交每项工作结果对应的工时，经项目经理审核后生效，计算其贡献值=工时*对应工作内容的权重，累计在各人名下。
        （二） 临时项目成员可以按照项目成员参与项目级别的分配。
        （三） 项目中途退出的项目成员与临时项目成员，除了要承担对项目造成的损失外，在本期分配时只能获得工时60%的贡献值，余下40%交给接手其任务的项目成员或临时项目成员，如有多个接手人则按本期接手人的贡献值比例分配。
        （四） 每次分配时，以项目成员的贡献值进行加权分配，各人分得盈余=总盈余*个人贡献值/项目贡献值总和。
        （五） 每次分配后贡献值调整的算法是：
    /// <summary>
    /// 贡献值数据
    /// </summary>
    public class ContributionData
    {
        public string PartnerName { get; set; }
        public string Job { get; set; }
        public double PreviousData { get; set; }
        public double CurrentData { get; set; }

        public ContributionData(string partnerName, string job, double previousData, double currentData)
        {
            PartnerName = partnerName;
            Job = job;
            PreviousData = previousData;
            CurrentData = currentData;
        }
    }

    public class Calculator
    {
        private Dictionary<string, double> PreviousData = new Dictionary<string, double>();
        private Dictionary<string, double> CurrentData = new Dictionary<string, double>();
        private List<ContributionData> Datas = new List<ContributionData>();

        #region 设置基础数据
        /// <summary>
        /// 设置上期贡献值
        /// </summary>
        /// <param name="job"></param>
        /// <param name="data"></param>
        private void AddPreviousData(string job, double data)
        {
            if (PreviousData.ContainsKey(job))
                PreviousData[job] += data;
            else
                PreviousData.Add(job, data);
        }

        /// <summary>
        /// 设置本期贡献值
        /// </summary>
        /// <param name="job"></param>
        /// <param name="data"></param>
        private void AddCurrentData(string job, double data)
        {
            if (CurrentData.ContainsKey(job))
                CurrentData[job] += data;
            else
                CurrentData.Add(job, data);
        }
        #endregion

        #region 输入
        /// <summary>
        /// 输入数据
        /// </summary>
        /// <param name="partnerName"></param>
        /// <param name="job"></param>
        /// <param name="previousData"></param>
        /// <param name="currentData"></param>
        public void AddData(string partnerName, string job, double previousData, double currentData)
        {
            ContributionData data = new ContributionData(partnerName, job, previousData, currentData);
            Datas.Add(data);
            AddPreviousData(job, previousData);
            AddCurrentData(job, currentData);
        }
        #endregion

        #region 计算工种的削弱度
        /// <summary>
        /// 计算工种的削弱度
        /// </summary>
        /// <param name="job"></param>
        /// <returns></returns>
        public double GetJobWeakenPercent(string job)
        {
            double jobPreviousData = PreviousData.ContainsKey(job) ? PreviousData[job] : 0;
            double jobCurrentData = CurrentData.ContainsKey(job) ? CurrentData[job] : 0;

            return jobPreviousData / (jobPreviousData + jobCurrentData);
        }
        #endregion

        #region 输出本期贡献值
        /// <summary>
        /// 输出本期贡献值
        /// </summary>
        public void PrintContributionList()
        {
            foreach (ContributionData item in Datas)
            {
                double contributionValue = item.PreviousData * GetJobWeakenPercent(item.Job) + item.CurrentData;
                Console.WriteLine(string.Format("Partner:{0},Job:{1},ContributionValue:{2}", item.PartnerName, item.Job, contributionValue));
            }
        }
        #endregion
    }

示例代码：
            Calculator cal = new Calculator();
            cal.AddData("李宠波", "程序员", 30, 10);
            cal.AddData("黄勇刚", "主程序员", 50, 0);
            cal.AddData("吴为问", "程序员", 0, 10);
            cal.PrintContributionList();


    二十三、 社员会议应制订《工作内容-权重对应表》，作为计算贡献值的权重。《工作-权重对应表》的制订过程如下：
        （一） 根据工作内容列出《工作内容清单》；
        （二） 根据各项工作内容的贡献大小排出顺序，可以并列（假设共有n个序号）；
        （三） 每次制订生成n组1~100随机数，每组有n个数字，最大值不超过最小值的五倍；
        （四） 社员会议每人一票，写明各组随机数符合集体利益的程度排序，按照n、n-1、…、1计分。总计得分最高的一组随机数自高到低与工作内容清单一一对应，成为一套《工作内容-权重对应表》。每套《工作内容-权重对应表》使用一次，社员会议应保持有4套以上备用，每次盈余分配时随机选出一套使用。
    二十四、 参与社员会议、项目会议的时间不计入工时。
    二十五、 合作社每个季度结束前十五天进行一次分配，平时可以由社员会议决定临时分配。


财务管理
    二十六、 专户：
        （一） 每名社员拥有一个对应的社员专户。
        （二） 合作社设立运营金、公积金、公益金三个专户，由管理委员会管理。
        （三） 每个项目设立项目专户、项目运营金、项目风险金三个专户，由项目经理管理。
    二十七、 社员专户：
        （一） 社员可以使用社员专户中的资金，也可以提取现金；
        （二） 提取现金时应与宿主单位签订劳务合同，税款由宿主单位代扣代缴；
        （三） 每季度结束前三个工作日应尽量提取现金，每年度结束前五个工作日应全部提取现金；
        （四） 项目经理只能在年度结束时提现。
    二十八、 合作社运营金：
        （一） 用于支付宿主单位管理费；
        （二） 用于支付合作社的产品运营、客户服务、工作设备、场地、资料、社员会议等支出；
        （三） 每季度第一个月末前，管理委员会有权从项目运营金专户提取下一季度末之前所需运营经费；
        （四） 新项目应在申报创立时在运营金专户中预留一个季度的运营经费；
        （五） 由管理委员会使用。
    二十九、 合作社公积金：
        （一） 用于加强合作社的持续能力。
        （二） 按照收入百分之五提取。
        （三） 由管理委员会使用。
        （四） 以公积金购买的资产为合作社资产，暂由宿主单位托管。
    三十、 合作社公益金：
        （一） 保障项目低谷期的社员。
        （二） 按照收入百分之五提取。
        （三） 由管理委员会使用。
        （四） 其中教育培训支出不得少于60%（百分之三）。
    三十一、 项目专户：
        （一） 项目设立时，项目成员应估算项目盈利前所需运转经费，从社员专户中转账到项目专户。
        （二） 各项收入扣除宿主单位管理费、合作社公积金、公益金之后，转入项目专户。
        （三） 项目经理应定期从项目专户调拨资金到项目运营金、项目风险金，使它们的余额符合要求。
        （四） 对立项登记中未注明分款规则的项目每季度末进行一次核算，在指定日期前将项目专户余额分配到项目成员的社员专户。
        （五） 对立项登记中注明分款规则的项目，按注明的分款日期进行一次核算，在指定日期前将分款数量（或比例）分配到项目成员的社员专户。
        （六） 当项目执行合同出现对外支付时，由管理委员会调用项目专户中的资金。
    三十二、 项目运营金：
        （一） 提供项目成员的工作条件所需经费，从项目运营金中提取；
        （二） 为了保证实现对委托方、消费者的承诺，在与委托方、消费者的协议签订之前，应把履行承诺所需的运营经费转入项目运营金专户，对委托方、消费者的服务承诺不得超出项目运营金所能服务的时间期限；
        （三） 项目停止运营时，项目运营金专户余额转入项目专户。
        （四） 项目运营金的默认提取规定为：
各项目平均分担未来一个季度应付的合作社运营金数量。根据项目的运营、客服等工作计划，管理委员会可以制定更高的提取规定。项目经理可以自愿多提取运营金。
    三十三、 项目风险金：
        （一） 项目风险金用于赔付项目相关合同的赔偿金，项目风险金应大于已签订合同的罚款总和，如金额不足则自动从项目专户拨入。
        （二） 当项目执行合同出现对外赔付时，由管理委员会调用项目风险金专户中的资金。
        （三） 当项目风险金不足以赔付时，由管理委员会按照次序调用以下专户：项目经理社员专户、项目专户、项目成员社员专户与临时项目成员专户、合作社公益金专户、合作社公积金专户、全体社员社员专户，前一类全部扣除完则调用下一类。
        （四） 当某项风险结束时，对应金额从项目风险金拨回项目专户。
    三十四、 每季度进行一次盈利核算，由管理委员会进行以下财务操作：
        （一） 将下一季度的运营经费从各项目运营金划到合作社运营金专户。
        （二） 剩余金额可以作为盈利，分配到参与项目的社员社员专户。
    三十五、 所有支出、报销、提现由管理员审核签字后，按照宿主单位财务程序办理。合作社内部转账由管理员批准后直接调整。所有专户余额和资金流动，由管理员维持账本，对所有社员公布。
        （一） 每一笔财务手续必须体现为内部转账或外部转账，不能出现无中生有的情形。必须有转出账户户主的签名，管理员才能接收办理。
        （二） 合作社内部专户之间转账：管理员检查转账单无误后，调整双方余额并书面通知双方。
        （三） 合作社对外转账：管理员检查转账单无误后，先调整转出方内部专户余额，然后填写外部转账表单，以书面形式并签名后交给宿主单位。宿主单位办理后把回单给管理员查阅，管理员应书面通知内部的转出方。
        （四） 到宿主企业办理报销，除了填写常规报销单之外，要填写对应的外部转账表单。
        （五） 个人专户提现，要填写每一笔对应的外部转账表单。每月汇总一次工资单给宿主企业，以便统一缴税。
        （六） 任何时候，社员可以根据管理员指定的方法查询：所有专户余额、所有内部转账表单、所有外部转账表单、合作社资金总表及流水账
        （七） 监事要定期检查内部转账表单、外部转账表单、合作社资金总表电子版和数字签名，确保它们之间的一致性。核实后向社员公布书面报告。
        （八） 相关表格见http://pm.itcoops.org:3000/attachments/download/21/admin.xls，管理员有权在其基础上增加字段、决定内容的格式、表现形式
    三十六、 无法按规定提取运营金的项目由合作社收回，由社员会议选举项目经理，原项目创始人的收益转入合作社公益金专户。

项目管理
    三十七、 社员引入委托开发项目应向管理委员会书面申请，经审核通过后称为项目创始人。提议新自营产品应向管理委员会书面登记，即可成为项目的创始人。
    三十八、 项目设一名项目经理，负责项目协作中的调度工作，由项目创始人任命。
    三十九、 委托开发项目
        （一） 委托开发项目需经过社员会议审核通过才可以立项。
        （二） 立项申请中可注明分款批次、日期（或项目进度要求）、数量（或比例），如未注明则按合作社默认规则进行分配。如果分款前提为项目进度要求，则项目进度要求由项目组成员每人一票超过半数的有效投票来决定，分款日期为通知管委会后的3个工作日内。
        （三） 立项申请中应注明创始人的回报方式（可多人共同申请，分别注明）：
            1、 由创始人提交源代码，并把各成员贡献值作为参量 运行程序得出各成员最终的贡献值
            2、 创始人源代码要求使用C#、 C++、 VB、java这四种语言之一编写，输入参数和输出格式由管委会公布
        （四） 创始人的回报是不可转让的、不可继承的。
        （五） 项目创始人参与实际工作时，因工作获得的权利与作为创始人获得的权利都有效。
    四十、 自营产品项目
        （一） 创始人可以选择回报方式，在立项登记时明确，有其它社员加入后不得更改：
            1、 由创始人提交源代码，并把各成员贡献值作为参量 运行程序得出各成员最终的贡献值
            2、 创始人源代码要求使用C#、 C++、 VB、java这四种语言之一编写，输入参数和输出格式由管委会公布
        （二） 立项登记时可注明分款批次、日期（或项目进度要求）、数量（或比例），如未注明则按合作社默认规则进行分配。如果分款前提为项目进度要求，则项目进度要求由项目组成员每人一票超过半数的有效投票来决定，分款日期为通知管委会后的3个工作日内。
        （三） 创始人的回报是终身的、不可转让的、不可继承的。
        （四） 项目经理拥有产品规划的决定权。
        （五） 自营产品的知识产权由合作社申请并持有，知识产权不得转让，知识产权授权由项目会议决定，收入扣除费用后转入项目专户。
        （六） 项目创始人参与实际工作时，因工作获得的权利与作为创始人获得的权利都有效。
    四十一、 通用规则
        （一） 项目经理：
            1、 确定分工，约定并登记各项目成员为项目保留的工时。
            2、 判定任务所属工作内容
            3、 审核并统计工时，送交管理委员会备案。
            4、 召集项目会议
            5、 起草合同，提交管理委员会审核，社员会议批准。
        （二） 项目成员：
            1、 与项目经理约定为项目保留的工时，一旦登记则有义务投入工作；
            2、 及时更新六个月内各周可投入工时。
    四十二、 项目归属
        （一） 当合作社分立时，由引入该项目的社员（委托开发项目）或者创始人（自营产品项目）决定加入哪一方，如果引入该项目的社员或者创始人已经不是合作社社员，则由项目会议表决决定，如果项目已经不再缴纳管理费则由社员会议决定。

宿主单位
    四十三、 财务行政管理
        （一） 发票：开发票的收入，收取8%作为税款。如实际经营的税负超过收入的8%，宿主单位出示纳税证明后根据实际比率补齐。
        （二） 社员专户提现：个人所得税按照法规代扣代缴，宿主单位应提供总数吻合的完税证明。
        （三） 项目支持：提供svn、zentao等服务器。
        （四） 办公：提供办公场地以及网络、打印等基本设备。
        （五） 记账：合作社相关专户由合作社管理委员会记账，宿主单位审核。
        （六） 财务行政服务的收费为每月3000元。
    四十四、 运营管理：如需单独的服务器、客户服务，按照当时实际成本估算费用并提前公布。
    四十五、 资产托管：合作社所拥有的资产名义上由宿主单位所有，相关费用由合作社承担。管理费用按照当时实际成本估算并提前公布。
    四十六、 知识产权托管：合作社的知识产权由宿主单位代为申请、管理，管理费用按照当时实际成本估算费用并提前公布。

    四十七、 本章程未尽事宜，由社员会议讨论决定。
    四十八、 本章程自签订之日起生效。
