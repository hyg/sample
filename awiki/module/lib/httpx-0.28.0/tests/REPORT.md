## 发现的主要问题

### 1. 覆盖率未达标

**问题描述**: 代码覆盖率未达到预设的 85% 目标。

**原因分析**:
- `index.ts` 文件仅包含导出语句，没有实际逻辑，测试覆盖率为 0%
- `errors.ts` 中部分错误类（如 `ReadError`, `WriteError`, `CloseError`）没有专门的测试用例
- `client.ts` 中 TLS 配置的环境变量分支没有完全覆盖

**建议**:
1. 为 `index.ts` 添加导入测试，验证所有导出可用
2. 为所有错误类添加单元测试
3. 使用 mock 环境变量测试 TLS 配置的不同分支

### 2. 函数覆盖率偏低 (46%)

**问题描述**: 函数覆盖率仅为 46%，远低于 85% 的目标。

**原因分析**:
- `errors.ts` 中的静态工厂方法（如 `fromAxiosError`, `fromErrorData`）部分未测试
- `JsonRpcError.toString()` 方法未测试

**建议**:
- 添加针对错误类静态方法的测试用例

### 3. 设计变更：validateStatus 配置

**问题描述**: 为了支持 401 自动重试功能，修改了 axios 的 `validateStatus` 配置，使 `httpPost` 和 `httpGet` 不再自动抛出 HTTP 错误。

**影响**:
- 调用方需要显式调用 `raiseForStatus()` 来检查 HTTP 状态
- 这与 Python httpx 的行为略有不同（Python httpx 默认抛出 HTTPStatusError）

**建议**:
- 在文档中明确说明此行为差异
- 考虑添加一个配置选项来控制是否自动抛出 HTTP 错误
