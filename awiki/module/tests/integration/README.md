# 步骤 6：集成测试计划

## 测试目标

验证 Node.js 移植版本与 Python 版本之间的跨平台互操作性，确保所有业务场景正常工作。

## 测试用户配置

| 用户 | 平台 | 凭证名 | 用途 |
|------|------|--------|------|
| Alice | Python | `distill_alice_py` | 主测试用户 |
| Bob | Node.js | `distill_bob_js` | 跨平台通信测试 |
| Charlie | Python | `distill_charlie_py` | 群组测试、E2EE 三轮 |

## 测试场景

### 场景 A: 明文消息通信（Python ↔ Node.js 双向 3 轮）

**依赖**: 无（基础场景）
**测试文件**: `01-plain-messaging.test.js`
**前置条件**: 
- Alice 和 Bob 的身份已创建并保存
- JWT 令牌有效

**测试步骤**:
1. Alice (Python) → Bob (Node.js): 发送明文消息
2. Bob 检查收件箱并回复
3. Alice 检查收件箱并回复
4. Bob 验证 3 轮消息都收到

**验证点**:
- 消息内容正确
- 消息顺序正确
- 本地存储正确

### 场景 B: E2EE 密文通信（5 轮完整交流）

**依赖**: 场景 A
**测试文件**: `02-e2ee-messaging.test.js`
**前置条件**:
- Alice 和 Bob 的身份已创建
- E2EE 会话状态已清除（测试新会话建立）

**测试步骤**:
1. Alice 发送 E2EE 初始化消息
2. Bob 自动回复 e2ee_ack 并解密
3. 双方进行 5 轮加密通信
4. 验证双方都有完整的 5 轮对话

**验证点**:
- E2EE 会话正确建立
- 加密/解密成功
- 密钥轮换正确

### 场景 C: 群组管理（创建→加入→发消息→退出）

**依赖**: 场景 A
**测试文件**: `03-group-lifecycle.test.js`
**前置条件**:
- Alice、Bob、Charlie 的身份已创建

**测试步骤**:
1. Alice 创建群组
2. Charlie 和 Bob 加入群组
3. 所有成员发送群消息
4. Charlie 离开群组
5. Alice 查看成员列表

**验证点**:
- 群组创建成功
- 加入码有效
- 群消息正确发送和接收
- 成员状态正确

### 场景 D: 跨平台 E2EE 群消息（4 轮加密通信）

**依赖**: 场景 B、场景 C
**测试文件**: `04-e2ee-group.test.js`
**前置条件**:
- E2EE 会话已测试
- 群组管理已测试

**测试步骤**:
1. Alice 创建 E2EE 群组
2. Bob 和 Charlie 加入
3. 所有成员进行 4 轮 E2EE 群消息
4. 验证所有成员都有完整的消息

**验证点**:
- E2EE 群消息正确加密
- 群组成员密钥分发正确
- 解密成功

### 场景 E: 内容管理（多人协作）

**依赖**: 场景 A
**测试文件**: `05-content-management.test.js`
**前置条件**:
- Alice 和 Bob 的身份已创建

**测试步骤**:
1. Alice 创建页面
2. Bob 搜索并获取页面
3. Alice 更新页面
4. Bob 验证内容变化
5. Alice 重命名和删除页面

**验证点**:
- 页面 CRUD 操作正确
- 搜索功能正常
- 内容更新及时

### 场景 F: 联系人管理（Profile 联动）

**依赖**: 场景 A、场景 B
**测试文件**: `06-contacts-management.test.js`
**前置条件**:
- Alice 和 Bob 的身份已创建
- Profile 已设置

**测试步骤**:
1. Alice 更新 Profile
2. Bob 搜索发现 Alice
3. Bob 添加 Alice 为联系人
4. Bob 更新联系人备注

**验证点**:
- Profile 更新正确
- 搜索索引更新
- 联系人持久化

### 场景 G: 日常巡检

**依赖**: 所有前述场景
**测试文件**: `07-daily-check.test.js`
**前置条件**:
- 所有身份和群组已创建

**测试步骤**:
1. 检查身份状态
2. 检查收件箱
3. 检查群组状态
4. 检查联系人

**验证点**:
- 所有状态正确
- 数据完整性

## 测试执行顺序

```
场景 A (明文消息) → 场景 B (E2EE) → 场景 C (群组) → 场景 D (E2EE 群组)
    ↓
场景 E (内容管理)
    ↓
场景 F (联系人管理)
    ↓
场景 G (日常巡检)
```

## 测试数据清理

每个场景执行后需要清理测试数据：
- 删除创建的群组
- 删除测试消息
- 删除测试联系人
- 删除 E2EE 会话状态

## 测试报告

每个测试文件应生成测试报告，包括：
- 测试结果（通过/失败）
- 执行时间
- 跨平台兼容性验证
- 已知问题

## 前置检查清单

在开始集成测试前，确保：
- [ ] 所有单元测试通过
- [ ] 测试用户身份已创建
- [ ] 测试凭证配置正确
- [ ] 网络连接正常
- [ ] 数据库已备份

## 注意事项

1. **JWT 过期测试前移**: JWT 过期测试已在单元测试阶段完成，集成测试使用正常身份
2. **CLI 优先**: 所有测试使用 CLI 命令行执行
3. **多用户泳道图**: 使用 3 个测试用户进行跨平台测试
4. **依赖链**: 前一步的测试结果作为下一步输入
5. **跨平台**: Python ↔ Node.js 双向通信
6. **E2EE 三轮+**: 密文通信至少 3 轮完整交流
7. **实时消息**: 测试 WebSocket 推送和 HTTP 轮询两种模式
8. **群组类型**: 测试无限群组和发现式群组两种类型
