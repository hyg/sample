有一个代码移植项目，需要把一个python产品移植到nodejs+windows环境。同时保持相似的用户体验。
我想使用coding agent去开发(coding agent的启动位置为本项目根目录)，但是项目规模超过AI上下文限制，导致AI在各个局部的代码无法集成。
下一步，我想用严格定义的步骤把项目切分为一系列小任务，以单个py文件为移植单位：
1. 下载和更新python产品的代码。获取最新代码的两个官方来源：
    1. **静态文件下载**（推荐，快速）
        - URL: `http://awiki.info/static-files/awiki-agent-id-message.zip`
        - 适用场景：快速下载最新版本
    2. **GitHub 仓库**（完整历史）
        - URL: `https://github.com/AgentConnect/awiki-agent-id-message`
        - 适用场景：查看提交历史、Issue 跟踪
2. 分析python代码，提取出：
2.1. 所有py文件和函数，种类（用户从命令行调用还是其它代码以函数方式调用。要严格、显式地区分脚本和代码，以cli调用的py文件使用bat文件去蒸馏，以函数方式调用的py文件使用py文件去蒸馏。对nodejs版本的测试同理。coding agent曾经以函数方式去“测试”cli接口，测试结果与用户体验不一致。）以及它们的互相调用关系，按照依赖关系划分移植次序，确保一个任务启动时所有依赖项已经移植完毕；
2.2. 依赖的外部包、版本号，分析本项目实际使用的接口函数；
2.3. 编写覆盖所有功能的测试脚本（注意和5.编写的单元测试区分）。
        - 多方互动的功能，安排 python、nodejs 版本的各种组合进行测试；
        - 多轮互动的功能，安排足够长度的多轮互动测试；
2.4. 根据python项目的许可证文件，选择本项目发行包的许可证，写入dist文件夹
2.5. 根据python项目的readme.md，结合本项目的情况编写发行包的readme.md，写入dist：
        - 不是官方开发，是用户个人开发
        - 作者不打算持续维护，没有任何承诺
        - 使用AI工具生成
        - 起草尽量完备的免责声明
3. 蒸馏这些py文件，提取输入输出值作为nodejs版本测试的基准(严格区分蒸馏和测试的用词。因为coding agent曾经在调用python版本代码后，以调试思路要去修改它认为测试不通过的“错误代码”。)：
3.1. 凡是用户从命令行调用的，就编写脚本以命令行方式蒸馏每个函数，记录输入输出值。
3.2. 凡是其它代码以函数方式调用的，就编写py代码以函数方式蒸馏每个函数，记录输入输出值。
3.3. 创建distill文件夹，在下面建立python版本同构、同名的文件夹结构，严格按照py文件的前缀命名蒸馏脚本或代码、蒸馏结果数据，保存在相同路径。依赖的外部包，创建专门的\distill\lib文件夹下为每个包创建一个子文件夹，子文件夹命名规则为：`<包名>-<版本号>`，例如：`httpx-1.0.0`，在专用文件夹内保存蒸馏脚本或代码、蒸馏结果数据。
4. 建立发行文件夹dist，在其中建立python版本同构、同名文件夹结构，以及lib子文件夹，严格按同路径、同前缀方式创建js文件：
4.1. python项目文件：`\python\a\b.py -> \dist\a\b.js`。
4.2. 依赖包文件：`\distill\lib\a.py -> \dist\lib\a.js`。
5. 准备：根据py代码、蒸馏数据以及代码种类，分别编写nodejs版本的单元测试脚本或者代码，以相同路径、相同前缀保存在\distill文件夹下。按照依赖关系创建移植任务保存为task.yaml：
5.1. 含有随机功能的函数，比对输出的格式和规则；
5.2. 不含随机功能的函数，严格比对输出值。
6. 移植，用严格的语言要求coding agent按task.yaml逐个任务完成，严厉地要求每次只完成规定的移植工作、忘记对其它任务的记忆：
6.1. 适配代码：python版本依赖的外部包在\dist\lib下开发，先根据 python 包功能寻找 npm 包：
        - 如果有功能相似的 npm 包，则安装、更新 package.json，然后在\dist\lib 文件夹内的 js 代码内容内使用。其它 js 代码导入\dist\lib 文件夹的 js 代码使用，接口、参数始终保持和 python 版调用外部包一致。
        - 如果没有功能相似的 npm 包，则根据功能自行实现。
    - 完成测试并编写测试报告。
6.2. 本项目代码编写代码并测试：严格沿用对应\python下的py文件的所有变量、函数、参数、返回值名称，按nodejs语法声明变量、函数；编写注释；对其它py文件的依赖关系也转移到对应的js文件。
7. 所有单元测试结束后，运行2.3.编写的脚本进行全面测试。根据遇到的问题，更新测试脚本或代码。
7.1. 编写.npmignore\package.json`文件写入dist文件夹。
8. 清理文件：coding agent会把带有错误理解的文档和代码导出保存，包括很深的子文件夹。这会影响后面AI的理解。而且用自然语言提示词无法清理干净，需要用nodejs开发专门工具清理所有文件夹，只保留以上工作流程必须使用的文件。

- 请根据以上设计skill.md文件，使用中文，参考规范：https://agentskills.io/specification
- 用到的代码工具（比如清理文件工具）分别设计发给codning agent的提示词。
- 涉及的数据结构都用yaml格式，每个都明确给出schema。
